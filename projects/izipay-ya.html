IziPay Ya – Integración de Pagos

Cliente: Interbank / NTT DATA · Año: 2023 · Rol: Backend Developer – Microservicios y Arquitectura

Contexto

Interbank buscaba modernizar y acelerar el procesamiento de transacciones digitales para su ecosistema de pagos con IziPay, permitiendo que comercios y usuarios pudieran realizar cobros en línea de forma más segura, estable y escalable.

El sistema previo presentaba problemas de latencia, baja observabilidad y alta dependencia entre componentes, lo que generaba retrasos en conciliaciones y poca trazabilidad en la auditoría de transacciones.

Solución Implementada

Se diseñó e implementó una plataforma basada en Arquitectura Hexagonal para desacoplar la lógica de negocio del proveedor externo IziPay, permitiendo un flujo de comunicación más estable y mantenible.

  • Microservicios REST para autorización, reversas y conciliación.
  • Adaptadores de infraestructura para comunicación directa con IziPay.
  • Persistencia flexible con MongoDB y caching con Redis.
  • Uso de Kafka para trazabilidad y auditoría en tiempo real.
  • Pipeline CI/CD en Azure DevOps totalmente automatizado.
  • Observabilidad avanzada con New Relic (APM, logs y trazas).

Al desacoplar la lógica de negocio y aplicar principios de Clean Architecture, se logró una solución más estable, escalable y fácil de extender.

Tecnologías Utilizadas

  • Java 17 – Spring Boot 2/3
  • Arquitectura Hexagonal / Ports & Adapters
  • MongoDB – Redis
  • Kafka – Event Streaming
  • Azure DevOps – Pipelines CI/CD
  • New Relic – Observabilidad
  • REST APIs – Integración de pasarelas de pago

Resultados Clave

  • Reducción del tiempo de procesamiento en más de 50%.
  • Disponibilidad cercana al 99.9%.
  • Trazabilidad completa del ciclo de vida de cada transacción.
  • Mayor resiliencia ante picos de carga.
  • Facilidad para integrar nuevos medios de pago a futuro.
